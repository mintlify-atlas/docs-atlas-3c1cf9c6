## Overview

The `Insertable` trait defines the contract for models that can be inserted into the database. It works with the `generic_insert` function to generate type-safe INSERT queries.

## Trait Definition

```rust
pub trait Insertable {
    const TABLE: &'static str;
    const COLUMNS: &'static [&'static str];

    fn bind<'q>(
        self,
        query: sqlx::query::QueryAs<'q, Postgres, Self, PgArguments>,
    ) -> sqlx::query::QueryAs<'q, Postgres, Self, PgArguments>
    where
        Self: Sized;
}
```

## Associated Constants

<ParamField path="TABLE" type="&'static str" required>
  The database table name (e.g., `"cities"`)
</ParamField>

<ParamField path="COLUMNS" type="&'static [&'static str]" required>
  Array of column names to insert (excluding auto-generated columns like `id`)
</ParamField>

## Required Methods

### `bind`

Binds the model's field values to the prepared query in the order specified by `COLUMNS`.

**Parameters:**
- `self` - Consumes the model instance
- `query` - SQLx QueryAs builder to bind values to

**Returns:** The query builder with all values bound

<Warning>
  The order of `.bind()` calls must match the order of columns in the `COLUMNS` array.
</Warning>

## Example Implementation

Here's how the `City` model implements `Insertable`:

```rust
impl Insertable for City {
    const TABLE: &'static str = "cities";
    const COLUMNS: &'static [&'static str] = &["name", "region"];

    fn bind<'q>(
        self,
        query: sqlx::query::QueryAs<'q, Postgres, Self, PgArguments>,
    ) -> sqlx::query::QueryAs<'q, Postgres, Self, PgArguments> {
        query
            .bind(self.name)
            .bind(self.region)
    }
}
```

## Usage with generic_insert

The `generic_insert` function uses this trait to generate and execute INSERT queries:

```rust
pub async fn generic_insert<T>(value: T, pool: &Pool<Postgres>) -> Result<T, Error>
where
    T: Insertable + for<'r> sqlx::FromRow<'r, sqlx::postgres::PgRow> + Send + Unpin,
{
    let columns = T::COLUMNS.join(", ");

    let placeholders = (1..=T::COLUMNS.len())
        .map(|i| format!("${}", i))
        .collect::<Vec<_>>()
        .join(", ");

    let query = format!(
        "INSERT INTO {} ({}) VALUES ({}) RETURNING *",
        T::TABLE,
        columns,
        placeholders
    );

    let q = sqlx::query_as::<_, T>(&query);
    let q = value.bind(q);

    q.fetch_one(pool).await
}
```

## Generated SQL

For the `City` model, `generic_insert` generates:

```sql
INSERT INTO cities (name, region) VALUES ($1, $2) RETURNING *
```

The trait implementation binds:
- `$1` = `self.name`
- `$2` = `self.region`

## Benefits

1. **Type Safety** - Compile-time guarantees that all columns are bound correctly
2. **Code Reuse** - Single `generic_insert` function works for all models
3. **Reduced Boilerplate** - No need to write SQL strings for each model
4. **Automatic RETURNING** - Always returns the inserted row with generated fields

## Adding to New Models

To make a new model insertable:

1. Define the struct with `#[derive(FromRow, Serialize)]`
2. Implement `Insertable` with table name and columns
3. Ensure `bind()` matches the column order
4. Use `generic_insert()` in your controller

<Tip>
  The `bind()` method consumes `self`, so you don't need to worry about lifetimes.
</Tip>
