Get started with Template API Rust - a minimal, modern REST API template built with Axum and PostgreSQL.

## Prerequisites

Before you begin, ensure you have:

- **Rust 1.83+** installed ([rustup.rs](https://rustup.rs))
- **PostgreSQL database** (local or remote like Supabase)
- **Docker** (optional, for containerized deployment)

## Installation

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/henvisi-1994/template_api_rust.git
    cd template_api_rust
    ```
  </Step>

  <Step title="Configure environment variables">
    Copy the example environment file and configure your database:

    ```bash
    cp .env.example .env
    ```

    Edit `.env` and set your database URL:

    ```bash
    DATABASE_URL=postgresql://usuario:contraseña@host:5432/base_de_datos?sslmode=require
    
    # Optional: Control logging verbosity
    RUST_LOG=info
    ```

    <Note>
      The `DATABASE_URL` must point to a valid PostgreSQL database. You can use a managed service like Supabase or a local PostgreSQL instance.
    </Note>
  </Step>

  <Step title="Run the application">
    <CodeGroup>
      ```bash Local Development
      cargo run --release
      ```

      ```bash Docker Compose (Recommended)
      docker compose up --build
      ```

      ```bash Docker Only
      docker build -t template-api-rust .
      docker run --rm -p 3000:3000 --env-file .env template-api-rust
      ```
    </CodeGroup>

    You should see output like:

    ```
    2026-03-11T10:30:15.123Z  INFO acquiring connection...
    2026-03-11T10:30:15.456Z  INFO connection acquired
    2026-03-11T10:30:15.457Z  INFO Server listening on http://0.0.0.0:3000
    ```
  </Step>

  <Step title="Test the API">
    Test the cities endpoint:

    ```bash
    curl http://localhost:3000/cities
    ```

    Expected response:

    ```json
    {
      "status": "success",
      "data": []
    }
    ```

    <Tip>
      The API is now ready! Check out the [Adding Resources](/guides/adding-resources) guide to learn how to create new endpoints.
    </Tip>
  </Step>
</Steps>

## Common Issues

<AccordionGroup>
  <Accordion title="Connection refused to database">
    Verify your `DATABASE_URL` is correct and the PostgreSQL server is running. Check network connectivity and firewall settings.
  </Accordion>

  <Accordion title="Compilation errors with SQLx">
    Ensure you have the correct SQLx version (0.6.2) and that your database is accessible during compilation if using compile-time checked queries.
  </Accordion>

  <Accordion title="Port 3000 already in use">
    The application binds to port 3000 by default. Change the port in `src/main.rs:53` or stop any services using that port.
  </Accordion>
</AccordionGroup>
